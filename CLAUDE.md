# CLAUDE.md

このリポジトリで作業するときのガイドです。

## このリポジトリは何か

国土数値情報（KSJ）を用いた空間分析・可視化の **活用事例集**。データ種別・テーマを問わず事例を追加していくドキュメント中心のリポジトリ（コードは各事例の外部リポジトリにある場合が多い）。記述は日本語。

## 構成と原則

```
cases/NN-slug/   分析事例（1事例 = 1ディレクトリ、自己完結）。README.md と images/ を持つ
common/          複数事例で再利用する共通コンポーネント
  road-network/  道路ネットワーク変換（「道路ネットワーク」テーマ事例の共通基盤）
data/            利用データの出典・入手方法のカタログ（実データ本体は置かない）
docs/            リポジトリ全体の共有ドキュメント・図
templates/case-template/  新規事例の雛形
```

- リポジトリは**特定データに依存しない汎用構成**。道路ネットワークは最初のテーマにすぎない。トップ README に道路固有の課題などを書かない（テーマ固有の話は各事例 or `common/` に置く）。
- 課題・制約はテーマ固有なら各事例 README、共通基盤の話なら `common/*/README.md` に書く。

## 新規事例の追加手順

1. `templates/case-template/` を `cases/NN-slug/` にコピー（`NN`=次の連番、`slug`=英小文字）
2. `README.md` を埋め、図を `images/` に配置
3. [`data/README.md`](data/README.md) のデータカタログに利用データを追記
4. トップ [`README.md`](README.md) の「分析事例一覧」表に1行追記
5. 道路ネットワークを使うなら `common/road-network/` を参照。新テーマで再利用可能な処理が出たら `common/` に切り出す

## コミットしないもの（.gitignore 済み）

- **GISデータ本体**（shp/geojson/gpkg/parquet/csv/zip 等）… 出典から入手する前提
- **投稿フォーム下書き**（`*投稿フォーム_下書き*.md`）… 国土数値情報「活用事例ページ」投稿フォーム用のローカル下書き。**氏名・メール等の個人情報プレースホルダを含むため公開リポジトリには絶対に含めない**

## 投稿フォーム下書きについて

`国土数値情報_活用事例ページ投稿フォーム_下書き_道路データ.md`（ローカルのみ）は活用事例ページへの投稿内容。リポジトリの各 README と内容（特に課題記述・使用ソフトウェア・デモのラベル）を**整合させる**こと。片方を直したらもう片方も確認する。

## 用語・表記の統一

- デモのラベル: 「到達圏分析・経路探索デモ」
- 速度の課題: 「速度設定の精緻化（道路分類種別）」
- 使用ソフトウェア: Python、MapLibre GL JS
