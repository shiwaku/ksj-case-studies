# 道路ネットワーク変換

国土数値情報 道路データ（N13, ラインデータ）を、経路探索に利用できる**ノード・リンク形式の道路ネットワーク**へ変換する共通基盤です。「道路ネットワーク」テーマの事例（[`cases/01`](../../cases/01-reachability-routing/)〜[`04`](../../cases/04-tsunami-evacuation/)）がこのネットワークを土台に利用します。

使用ソフトウェア: Python（ネットワーク変換・経路探索の独自実装）、MapLibre GL JS（可視化）、QGIS（可視化）。

## 処理の流れ

1. **ノード・リンク生成**
   道路データ（LineString）の各ラインの始点・終点座標を抽出し、同一座標に「**2次メッシュコード6桁＋メッシュ内連番5桁**」の11桁ノードIDを付与してノード・リンクを生成。

2. **リンク長・所要時間の算出**
   - リンク長: **Haversine公式**で頂点間距離を積算。
   - 所要時間: 道路分類種別ごとの速度（国交省「都市交通計画検討のための手引き」の自由旅行速度を根拠）から算出。

3. **双方向化・出力**
   一方通行のフィールドが存在しないため、全リンクを**双方向**で出力。**CSV** および **GeoParquet（EPSG:6668）** 形式に整形。

4. **アクセスリンク作成**
   メッシュ重心・鉄道駅・バス停を最寄りの道路ノードへ接続するアクセスリンクを作成。その際、高速本線上のノードや主要道路と接続していない孤立ノードはスナップ対象から除外し、経路の行き止まりを防止。

## 経路探索

- ダイクストラ法
- マルチソースダイクストラ法（複数始点からの到達圏分析）

## 既知の制約

- 一方通行が未考慮（全リンク双方向）
- 高速自動車国道と都市高速道路の区別ができない
- 速度設定の精緻化（道路分類種別）が未対応
